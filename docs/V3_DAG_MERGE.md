# v3 DAG и merge-коммиты

## Линейная миграция (сейчас)

`tc migrate-v3 --apply` строит цепочку, где у каждого `Commit` ровно **0 или 1** родитель (`parents[]`).

## Merge (следующий шаг)

Когда два subagent-а сходятся:

- новый `Commit` с `parents: [digest_A, digest_B]` и одним общим `tree`;
- `tc log --graph` расширяется до отображения join-узла (`|\\|`).

Формат хранения тот же CAS JSON; меняется только массив `parents` и политика verify (топологическая сортировка).
